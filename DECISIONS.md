# raster-forge — DECISIONS

Архитектурные и проектные решения с обоснованием.

| Дата | Решение | Альтернативы | Обоснование |
|---|---|---|---|
| 2025-12-30 | Язык: Go | Rust, Python, Node.js | Целевой домен — утилита командной строки с интенсивным I/O и параллелизмом. Go даёт один бинарник, статическую линковку, горутины и приемлемую производительность без сложности Rust. |
| 2025-12-30 | Конфигурация: YAML | JSON, TOML, HCL | YAML — де-факто стандарт для конфигов в Go-экосистеме, поддерживает комментарии, человекочитаем. |
| 2025-12-30 | Пирамида: от большего к меньшему | Каждый уровень независимо | Кэширование: downsample с предыдущего уровня, а не каждый с оригинала. O(N) вместо O(N²) по времени. |
| 2025-12-30 | WebP как формат пирамиды | PNG, JPEG | WebP даёт лучшее сжатие с lossy (для пирамиды) и lossless (для архивации). Поддерживается всеми VLM. |
| 2025-12-30 | Растрированный PDF + пирамида | Только пирамида, только PDF | PDF нужен как fallback для инструментов, не поддерживающих WebP. Пирамида — для VLM. |
| 2025-12-30 | Выходной каталог с timestamp | Фиксированное имя | Предотвращает перезапись предыдущих результатов. |
| 2025-12-30 | `MaxGoroutines: 0` = NumCPU | Всегда фиксированное число | Дефолт «используй все ядра» — ожидаемое поведение для CLI-утилиты. |
| 2026-07-16 | JSON-режим для интеграции | Только файловый вывод | DocuMind должен вызывать raster-forge как subprocess и читать результат без парсинга stdout. |
| 2026-07-16 | Атомарная запись через tmp+rename | Прямая запись | Гарантирует, что частично записанный файл не будет прочитан DocuMind. |
| 2026-07-16 | Pipeline stages как интерфейсы | Одна функция | Позволяет тестировать каждую стадию независимо и заменять реализации (pdfcpu ↔ MuPDF). |
